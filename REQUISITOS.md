Quero que crie o ABCode, um agente de codificação que recebe uma demanda e cria um plano de execução para essa demanda. Um loop de interação com o usuário deve ser estabelecido com o modelo pedindo mais informações ao usuário se achar necessário dirimir dúvidas. Quando o usuário dizer que o plano está ok, o agente transforma esse plano em subplanos (subetapas gerenciaveis para modelos pequenos) e então vai executando as subetapas uma a uma. Se for necessário mais informações em uma subetapa, pode-se sempre pedir ao usuário mais informações (pode usar o padrão use tools para isso). Se alguma subetapa falhar, o agente deve tentar o máximo de três vezes (por padrão, mas deixando um parâmetro para que se possa aumentar isso) com alternativas. Se o agente não conseguir uma etapa, deve informar ao usuário, notificando os erros. O sistema deve funcionar como cli de outros agentes, mostrando o pensamento e as etapas intermediárias com efeitos de terminal elegantes. O sistema deve ser refinado para suportar principalmente modelos pequenos e menos autônomos, mas o agente como um todo deve parecer autônomo. A arquitetura deve ser modularizada e elegante. Pode ignorar o código existente no interactiveplanner.py (apesar de poder usá-lo para aprender a biblioteca AgenticBlock.io). Observe que você deve usar o padrão MemGPT da forma já implementada e suportada na AgenticBlocks.IO (consulte o projeto para ver como funciona). A interação como o usuário deve ser me modo de sessão. Quando o usuário executa o sistema, é criada uma sessão, toda informação da seção deve ser salva no banco de dados do MemGPT. O usuário deve fornecer um nome de seção assim que abre o sistema. Se for um nome de seção existente no bd, o sistema deve perguntar se ele quer abrir uma seção existente ou se quer criar uma nova e sobreescrever a existente. O sistema deve mostrar os planos em linguagem natural, mas o planejamento interno deve ser em código, usando a biblioteca python e qualquer módulo que permita execução no sistema subjacente. O sistema deve ter um modo de execução: auto (faz tudo sem pedir permissão ao usuário), plan (crie um plano e pede aprovação do usuário), edit (cria o plano e pedi permissão do usuário apenas para cada operação sensível - alterar sistemas de arquivos, enviar dados na rede, criar ou apagar usuário, etc).
