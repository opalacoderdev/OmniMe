# 08 — IDE Integration

This document specifies the integration architecture between **OpalaCoder** and the **OpalaCoderIDE** (React-based GUI). It describes the execution modes, communication protocols, and APIs that bridge the agent engine with the web client.

---

## 1. Execution Modes

OpalaCoder supports two main entry points for third-party and IDE integrations:

1. **Agent Stdin/Stdout Protocol (`--stdin`)**: Starts a persistent loop listening for JSON-formatted commands on standard input and printing structured JSON events to standard output.
2. **React Web GUI & REST API (`--gui`)**: Launches an asynchronous HTTP server serving a single-page React application and exposing a backend REST API. This mode internally wraps the agent execution engine and streams agent events to the browser.

---

## 2. Stdin/Stdout Agent Protocol (`agent_stdin.py`)

When started via `python main.py --stdin`, the program redirects standard Python `stdout` prints to `sys.stderr` to prevent stream pollution, keeping `sys.stdout` exclusively for JSON event communication.

### Inbound Commands (Stdin)

Commands are sent as single-line JSON objects terminated by a newline.

#### `load_project`
Loads or initializes a project context.
```json
{
  "command": "load_project",
  "project_name": "my_project",
  "project_path": "/absolute/path/to/project",
  "db": "/path/to/sessions.db"
}
```

#### `run`
Executes an agent block or a CLI command.
```json
{
  "command": "run",
  "agent": "chat_orchestrator",
  "prompt": "Create a python script that prints hello",
  "model": "ollama/gemma4",
  "system_prompt": "Optional system prompt override",
  "messages": [],
  "tools": "all"
}
```
*Note: If the `prompt` starts with a slash `/` (e.g., `/help`, `/undo`), the server intercepts and dispatches it directly to the CLI command registry.*

#### `list_projects`
Retrieves all persisted projects.
```json
{
  "command": "list_projects",
  "db": "/path/to/sessions.db"
}
```

#### `create_project`
Registers a new project in the SQLite database.
```json
{
  "command": "create_project",
  "project_name": "My New Project",
  "project_path": "/path/to/project",
  "description": "Optional description",
  "model": "ollama/gemma4",
  "mode": "auto",
  "skills": ["chat-orchestrator"]
}
```

#### `delete_project`
Deletes a project.
```json
{
  "command": "delete_project",
  "project_name": "my_project"
}
```

#### `exit`
Terminates the stdin protocol loop.
```json
{
  "command": "exit"
}
```

### Outbound Events (Stdout)

Events are serialized as single-line JSON objects written directly to stdout.

- `server_ready`: Emitted when the stdin server loop is initialized.
- `project_loaded`: Confirms a project has been successfully loaded.
- `project_created`: Confirms new project registry.
- `projects_list`: Returns the list of projects from the DB.
- `agent_started`: Indicates that an agent execution cycle has begun.
- `thought`: Streams step-by-step thoughts or reasoning steps from the LLM.
- `tool_call`: Emitted when an agent invokes a tool (e.g., `read_file`, `run_command`).
- `tool_result`: Returns the outcome/payload of a tool invocation.
- `agent_response`: The final response text generated by the agent.
- `agent_finished`: Indicates completion of the execution cycle.
- `error`: Reports exceptions, tracebacks, or failed commands.

## 3. IDE Server and REST API (`ide_server.py`)

Started via `python main.py --gui`. It spawns a custom, lightweight Python `AsyncHTTPServer` (built on top of `asyncio.start_server`) that hosts static files and provides a REST API.

### Static File Server

The server resolves and serves the built single-page application (SPA) assets from the `gui/` directory inside the package. It defaults to serving `index.html` for any unmatched path, supporting client-side routing fallback.

### API Endpoints

All API routes are prefixed with `/api/`.

#### `GET /api/files?projectPath=<path>`
Recursively scans the directory and returns a sorted hierarchical tree of files and directories (excluding directories like `node_modules`, `.git`, `.venv`, and `__pycache__`).

#### `GET /api/file/read?projectPath=<path>&filePath=<rel_path>`
Reads and returns the text contents of the requested file. Path traversal is prevented by validating that the resolved path stays within the `projectPath` boundaries.

#### `POST /api/file/write`
Saves updated content back to the file.
- **Body**: `{"projectPath": "...", "filePath": "...", "content": "..."}`

#### `GET /api/opalacoder/list-projects`
Fetches all stored projects from `ProjectStore`.

#### `POST /api/opalacoder/create-project`
Creates a project entry in the database.
- **Body**: `{"project_name": "...", "project_path": "...", "description": "...", "model": "...", "mode": "...", "skills": []}`

#### `POST /api/opalacoder/delete`
Removes a project from the DB store.
- **Body**: `{"project_name": "..."}`

#### `POST /api/opalacoder/run`
Runs an agent session using **Chunked Transfer Encoding** to stream JSON events to the browser in real-time.
- It attaches a `web_event_hook` to `agent_stdin.event_hook`.
- While the background agent task runs via `handle_run(data)`, any events emitted to the hook are pushed to an `asyncio.Queue` and flushed out to the client as raw chunked HTTP frames.

## 4. UI Asset Merging and Building

1. **Source (`gui_src/`)**: The React codebase sits in `gui_src/` (configured via `vite.config.js`).
2. **Build and Integration**:
   - `npm run build` compiles the React codebase into static production files in a `dist/` directory.
   - `copy_gui.py` cleans `opalacoder/gui/` and copies the compiled assets from `dist/` into it, embedding the frontend assets within the Python distribution package.
